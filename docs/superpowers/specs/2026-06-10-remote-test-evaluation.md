## 一、Remote Test 方案原文（可直接插入文档）

### 4. Remote Test 方案

#### 4.1 测试设计

| 项目 | 内容 |
|------|------|
| 用户数量 | 3–5名/轮 |
| 测试时长 | 45–60分钟/人 |
| 测试类型 | Remote Moderated（主持型）+ Remote Unmoderated（非主持型）结合 |
| 观察方式 | 屏幕共享 + 语音同步，主持人实时引导 + 观察员记录 |

>⚠️ **补充说明（待加入）：** 考虑到中美时差，每轮测试安排在北京时间 21:00–22:00（美国东部上午 9–11 点）或双方协商确定，单次时长控制在 45 分钟内。

#### 4.1.5 用户招募（待补充）

| 项目 | 内容 |
|------|------|
| 招募渠道 | Maze/UserTesting 内置用户库，或自建招募池 |
| 筛选条件 | 例：iOS 用户、最近 30 天使用过竞品、特定职业/背景 |
| 激励预算 | 美国用户市场价：45 分钟测试 $75–$150 Amazon Gift Card |
| 联系工具 | 邮件/邀请链接，需提前告知测试平台和要求 |

#### 4.2 工具选型

##### 4.2.1 主持型测试工具（Moderated）

| 工具 | 优点 | 缺点 | 适合场景 | 推荐度 |
|------|------|------|----------|--------|
| **Lookback** | 专为 UX 研究设计，支持录屏 + 标注 + 实时评论 + 回放；用户只需打开链接，无需下载 App | 成本较高；部分用户需下载 App 或插件 | 系统化 UX 团队深度访谈、长期研究 | ⭐⭐⭐⭐⭐ |
| **Zoom** | 最稳定、用户最熟悉、企业环境可用；支持观察室模式；录屏方便 | 免费版 40 分钟限制；无 UX 专项标注功能；需团队翻墙 | 正式主持测试、观察员同步观看；作为 Lookback 的备份方案 | ⭐⭐⭐⭐ |
| **Google Meet** | 免安装、浏览器直接进入；Google 生态集成；国内可直接使用 | 录屏功能较弱；无专门 UX 研究功能 | 低门槛海外用户招募、快速测试；VPN 不稳定时的备选 | ⭐⭐⭐ |

##### 4.2.2 非主持型测试工具（Unmoderated）

| 工具 | 优点 | 缺点 | 适合场景 | 推荐度 |
|------|------|------|----------|--------|
| **Maze** | 任务路径测试强；转化漏斗可视化；适合分析"哪里掉人"；支持图片/原型测试 | 偏向任务型测试；无法追问；非完整用户体验观察 | 验证交互流程、转化率优化、大规模初步验证 | ⭐⭐⭐⭐ |
| **UserTesting** | 快速、便宜（相对）、规模化；全球用户库大；可批量测试 | 不能追问、只能"看录像猜"；洞察深度有限；价格实际不便宜 | 大规模定量验证、初步概念测试、A/B 测试 | ⭐⭐⭐ |

#### 4.3 价格对比

| 工具 | 定价模式 | 参考价格/月 | 备注 |
|------|----------|-------------|------|
| **Lookback** | 按席位 | $99/人/月起 | 2024 年上调过，早期报价 $75 已不适用 |
| **Zoom Pro** | 按账号 | $14.99–$19.99/人/月 | 会议40 分钟限制；Business Plus ~$20.99，含云录制 |
| **Google Meet Pro** | 按账号 | $8–$12/人/月 | 基础功能可用，录制需 Plus 版 |
| **Maze** | 按席位 | $32–$64/人/月 | 有免费额度；适合中型团队 |
| **UserTesting** | 按测试次数 | $299–$499/次 | Live Conversation $399/次起；适合定量验证 |

> 📌 注：以上价格为 2024 年市场参考价，实际以各平台官网最新报价为准。

#### 4.4 推荐工具组合方案

##### 方案A：先难后易（快速验证 + 深度挖掘）

**适合场景：** 团队需要快速积累数据、先出初步结果给领导汇报

**工具组合：**
- 主持型：Zoom（主力）+ Lookback（深度场次备用）
- 非主持型：Maze（先跑一轮大规模验证）

**流程：**
1. **第一轮（Maze，非主持）：** 快速触达美国用户，收集任务完成率数据，找到"哪里掉人"
2. **第二轮（Zoom，主持型）：** 基于 Maze 数据，选取典型用户深入追问，挖掘问题根源
3. **第三轮（Lookback，主持型）：** 重点场次用 Lookback 录制标注，形成深度案例

**优势：** 前期成本低、速度快，能快速出数据
**劣势：** 需要两轮测试才能拿到深度洞察

##### 方案B：直接深入（质量优先）

**适合场景：** 追求洞察质量，愿意花时间做好每一轮，不急于出结果

**工具组合：**
- 主持型：Lookback（主力）+ Zoom（备用）
- 非主持型：Maze（辅助）+ UserTesting（大规模补充验证）

**流程：**
1. **每轮测试（Lookback）：** 专人引导、录屏标注、实时追问，每轮都是深度案例
2. **穿插 Maze 测试：** 验证特定任务路径，量化转化漏斗
3. **季度性 UserTesting：** 大规模用户库补充定量数据

**优势：** 每一轮都拿到高质量洞察，UX团队成长快
**劣势：** 前期成本相对较高，用户端需要适应 Lookback

#### 4.5 方案对比总结

| 对比维度 | 方案A（先难后易） | 方案B（直接深入） |
|----------|-------------------|-------------------|
| **工具重心** | Zoom + Maze | Lookback + Maze |
| **洞察深度** | 中等（需多轮积累） | 高（每轮都深） |
| **速度** | 快（1–2 轮见效） | 慢（稳步推进） |
| **前期成本** | 低 | 中高 |
| **适合阶段** | 快速验证、初步探索 | 深度研究、长期体系 |
| **团队要求** | 中等（Zoom 易上手） | 较高（Lookback需学习） |

#### 4.6 数据分析与输出（待补充）

| 项目 | 内容 |
|------|------|
| 整理节奏 | 每轮测试后 24–48 小时内整理录制和笔记 |
| 分析工具 | Lookback 内置标注；Maze 后台自动生成漏斗数据 |
| 洞察提炼 | 使用 Nielsen Norman 问题严重度评级（P0/P1/P2）分类 |
| 输出物 | 每轮测试出具《用户洞察报告》，包含：用户行为摘要、问题列表、建议优先级 |
| 迭代衔接 | 洞察报告结论直接进入下一轮测试设计，形成闭环 |

## 二、方案问题清单（评估结论）

### 问题1：中美时差 — 根本约束未写入方案 🔴

| 中国时间 | 美国东部(EST) | 美国西部(PST) |
|----------|---------------|---------------|
| 9:00 | 前一天 20:00 | 前一天 17:00 |
| 21:00 | 9:00 | 6:00 |

**实际情况：**
- 适合双方的时间窗口：北京时间 **21:00–23:00**（美国东部上午 9–11 点）
- 45 分钟是单次极限，否则要么团队熬夜，要么用户早起
- 超过1 小时 session 必须牺牲某一方

**建议：** 在 4.1 测试设计里加一句时差说明（已体现在上方原文的补充说明中）

---

### 问题2：Lookback 用户端下载门槛被低估 🔴

- 美国普通用户收到测试邀请，被告知「请下载 Lookback App」→ **直接流失**
- Maze / UserTesting / Zoom 都是点链接就能测，用户零摩擦
- Lookback 只有在专门招募**已有 Lookback 经验的专业参与者池**时才顺畅

**建议：**
- 第一轮不用 Lookback，先用 Zoom（点链接即用，用户最熟悉）
- Lookback 作为后续进阶，或专门针对有 Lookback 经验用户使用时引入

---

### 问题3：用户招募渠道和筛选条件缺失 🔴

Remote test 最难的不是工具，是**人从哪来**。方案没有提到：

| 激励多少 | 美国用户市场价：45 分钟测试 $75–$150 Amazon Gift Card |
| 如何联系 | 邮件？邀请链接？飞书？ |

**建议：** 在 4.1 后增加「4.1.5 用户招募」小节（已体现在上方原文）

---

### 问题4：Zoom 在中国大陆的可用性 🟡

- 团队在中国，Zoom 在国内需要**翻墙**才能稳定使用
- 免费版国内可直接用，但有 40 分钟限制
- 商务版需要翻墙，且企业账号需要境外公司主体

**建议：** 
- 如果 VPN 稳定 → Zoom 可用
- 如果 VPN 不稳定 → **Google Meet**（国内可直接用）是更安全的备选
- 方案里注明「稳定的团队 VPN 环境」是使用 Zoom 的前提条件
