# 数据爬取底层逻辑说明

## 🔄 整体流程

```
数据源网站 → crawl.py爬虫 → articles.json → GitHub仓库 → GitHub Pages发布 → 用户访问
                                    ↓
                          improve_summaries.py
                          (DeepSeek-V3生成摘要)
```

##📁 核心文件

### 1. crawl.py（爬虫主脚本）
- 职责：从各数据源网站抓取原始文章
- 运行环境：本地或GitHub Actions
- 输出：原始文章数据（含英文标题和摘要）

### 2. improve_summaries.py（摘要增强脚本）
- 职责：调用DeepSeek-V3 API生成中文摘要
- 输入：crawl.py输出的原始数据
- 输出：带中文标题、一句话总结、why_important、how_to_use的完整数据

### 3. articles.json（最终数据）
- 格式：JSON数组，每条记录包含完整字段
- 路径：`data/articles.json`
- GitHub Pages访问路径：`/marketing-intelligency/data/articles.json`

##🕷️ 爬虫逻辑 (crawl.py)

```
1. 初始化配置
   ├── 定义数据源列表（RSS订阅源、目标网站）
   └── 设置爬取频率和超时

2. 遍历数据源
   ├── 获取RSS Feed或网页内容
   ├── 解析HTML/XML提取文章信息
   └── 字段提取：
       ├── title：文章标题
       ├── url：文章链接
       ├── published：发布时间
       ├── source：来源标识
       └── department：所属部门（自动分类）

3. 数据去重
   ├── 按URL去重
   └── 按标题去重（模糊匹配）

4. 输出原始JSON
   └── 保存到临时文件或直接传递给下游
```

## 🤖 摘要增强逻辑 (improve_summaries.py)

```
1. 读取原始文章数据

2. 批量调用DeepSeek-V3 API
   提示词模板：
   """
   请为以下文章生成中文摘要：
   标题：{title}
   原文摘要：{summary}

   请生成：
   1. title_cn：中文标题
   2. one_sentence：一句话总结（15字内）
   3. why_important：为什么重要（针对跨境电商定制礼品行业）
   4. how_to_use：怎么用（Callie.com实操建议）
   5. importance：重要程度（高/中/低）
   """

3. 解析API响应并更新文章数据

4. 输出完整articles.json
```

## 📊 字段生成规则

### importance（重要性）评分标准
| 等级 | 评判维度 | 示例 |
|------|----------|------|
| 高 | 政策变化、算法更新、市场突变、与Callie核心业务强相关 | Google AI搜索重大更新、定制礼品关税政策变化 |
| 中 | 行业趋势、新工具发布、有一定参考价值 | 新型社交媒体分析工具发布 |
| 低 | 边缘内容、时效性弱、相关性低 | 一般性行业新闻 |

### department（部门）自动分类
| 关键词 | 归类 |
|--------|------|
| brand, branding, brand strategy | 品牌 |
| seo, search, google, bing | SEO |
| ads, advertising, ppd, sem | 广告投放 |
| user, customer, research, survey | 用户洞察 |
| competitor, competitor analysis | 竞品最新动态 |
| social media, facebook, instagram | 社交媒体运营 |
| trending, viral, popular content | 社媒热门内容 |
| custom, promotional, gift, OEM | 定制礼品最新行业趋势 |

## 🌐 GitHub Actions 自动部署

```yaml
# .github/workflows/crawl.yml概要
on:
  schedule:
    - cron: '0 2 * * *'  # 每天凌晨2点执行

jobs:
  crawl:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Python
        uses: actions/setup-python@v5
      - name: Install dependencies
        run: pip install requests feedparser beautifulsoup4
      - name: Run crawler
        run: python crawl.py
      - name: Generate summaries
        run: python improve_summaries.py
      - name: Commit and push
        run: |
          git config user.name "GitHub Actions"
          git add data/articles.json
          git commit -m "chore: auto-update articles $(date +%Y-%m-%d)"
          git push
```

## ⚠️ 注意事项

### URL路径问题
- GitHub Pages从`docs/`目录serve
- 正确fetch路径：`data/articles.json`
- 错误写法：
  - `/marketing-intelligency/data/articles.json`（绝对路径，指向根目录）
  - `../data/articles.json`（上级目录，指向repo根）

### Hugo部署冲突
- Hugo的`--destination`会删除docs/已有文件
- 部署前需备份articles.json
- 或使用增量更新策略

### 数据源优先级
1. RSS Feed（结构化、可靠）
2. 目标网站HTML抓取（需解析）
3. 第三方API（如BuzzSumo，需付费）
