# callie.com 营销资讯中心 使用手册

## 📖 功能概览

营销资讯中心聚合8大版块的行业资讯，帮助营销人员快速获取有价值的信息。

## 🎯 8大版块

| 版块 | 说明 |
|------|------|
| 品牌 | 品牌建设与维护相关资讯 |
| SEO | 搜索引擎优化技巧与趋势 |
| 广告投放 | 在线广告投放策略与案例 |
| 用户洞察 | 用户行为分析与市场研究 |
| 竞品最新动态 | 竞争对手动态监控 |
| 社交媒体运营 | 社交媒体运营策略 |
| 社媒热门内容 | 热门内容趋势分析 |
| 定制礼品最新行业趋势 | 定制礼品行业最新趋势 |

## ⭐ 重要程度星级评分标准

| 星级 | 重要性等级 | 评分标准 |
|------|------------|----------|
| ★★★★★（5星） | 高 | 对Callie.com业务有重大影响，需优先关注，如：重大政策变化、颠覆性技术、市场突变 |
| ★★★☆☆（3星） | 中 | 对业务有一定参考价值，建议阅读，如：行业趋势报告、新工具发布 |
| ★☆☆☆☆（1星） | 低 | 一般性资讯，了解即可，如：常规更新、边缘性内容 |

**评分依据说明：**
- 由DeepSeek-V3模型根据文章内容自动判定
- 评判维度包括：与定制礼品行业的相关性、对跨境电商的潜在影响、内容时效性
- "高"级别文章会重点标注why_important（为什么重要）和how_to_use（怎么用）

##🔍 筛选功能说明

### 部门筛选（Tab栏）
点击不同Tab筛选对应部门的文章，"全部"显示所有文章。

### 内容类型筛选
- **趋势**：行业趋势分析报告
- **案例**：成功案例分享
- **教程**：操作指南
- **工具**：工具介绍与评测
- **数据**：数据报告、研究统计
- **观点**：专家观点、评论
- **其他**：不属于以上类型的文章

###重要程度筛选
- **重要**：★★★★★ 高重要性文章
- **一般**：★★★☆☆ 中等重要性文章
- **普通**：★☆☆☆☆ 低重要性文章

### 时间范围筛选
- **今天**：24小时内的文章
- **近7天**：7天内的文章
- **近30天**：30天内的文章

### 内容新旧筛选
- **🆕 新内容（近3个月）**：发布时间在90天内的文章
- **🔥 热门内容（近6个月）**：发布时间在180天内的文章
- 说明：用于区分近期发布的"新内容"和持续受欢迎的"热门内容"，两者可能有重叠

### 关键词搜索
输入关键词搜索文章标题、一句话总结、为什么重要、怎么用等字段。300ms防抖自动搜索。

## 📊 文章信息字段说明

| 字段 | 说明 |
|------|------|
| title | 原文标题（英文） |
| title_cn | 中文标题 |
| one_sentence | 一句话总结 |
| why_important | 为什么重要（针对Callie.com） |
| how_to_use | 怎么用（针对Callie.com的实操建议） |
| published | 文章原始发布时间 |
| crawled_at | 爬取时间 |
| source | 来源标识 |
| content_type | 内容类型 |
| department | 所属部门 |
| importance | 重要性等级 |

##⏰ 更新时间与存档逻辑

### 数据更新频率
- **自动更新**：每日由 GitHub Actions 自动爬取一次
  - **GitHub Actions 使用 UTC 时区**
  - 当前设置：`0 1 * * *` = **UTC 凌晨 1:00**
  - 换算为中国时间（UTC+8）：**上午 9:00**
- **手动更新**：管理员可通过触发 GitHub Actions 手动执行爬取

### 文章发布时间显示
- 每篇文章显示📅**发布时间**（`published`字段）
- 优先级：`published`（原文发布时间）> `crawled_at`（爬取时间）
- 如果两者都没有，则不显示时间标签

### 时间范围筛选逻辑
| 筛选条件 | 计算方式 | 说明 |
|----------|----------|------|
| 今天 | 文章发布时间距今 < 24小时 | 包含今天发布的文章 |
| 近7天 | 文章发布时间距今 ≤ 7天 | 7天内（含第7天） |
| 近30天 | 文章发布时间距今 ≤ 30天 | 30天内（含第30天） |

### 数据存档策略
- **数据文件**：`data/articles.json` 包含全量历史数据（不删除旧文章）
- **文件位置**：存放于 GitHub 仓库 `data/articles.json`
- **版本控制**：通过 Git 记录每次更新，支持回滚历史版本
- **保留周期**：目前为全量保留，暂无自动清理机制

### 文章去重规则
爬虫执行时按以下规则去重：
1. **URL去重**：相同URL的文章只保留最新版本
2. **标题去重**：标题完全相同的文章视为重复

### 最后更新时间
- 页面底部显示"最后更新: XXXX年XX月XX日 XX:XX"
- 该时间取自数据集中最新文章的 `crawled_at` 字段
- 用于判断当前数据的新鲜程度
