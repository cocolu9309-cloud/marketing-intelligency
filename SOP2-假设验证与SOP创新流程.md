# SOP 2：假设验证与 SOP 创新

## 1. 目的

建立营销工程师从假设提出到验证落地全流程的标准方法，确保每一个新玩法、创新优化都能通过系统化的验证，最终固化为可复用的 SOP 交付给团队。

## 2. 适用范围

营销工程师主导的课题研究与 SOP 创新升级工作

## 3. 课题来源

营销工程师的课题来源分为四类：

| 来源类型 | 说明 | 发现频率 |
|---------|------|---------|
| 运营数据异常 | 从后台数据中发现转化率、点击率等指标异常 | 随时 |
| 竞品观察 | 通过竞品动态或行业案例发现新的可能性 | 每周 |
| 行业报告 | 阅读行业报告时发现值得验证的假设 | 每月 |
| 个人测试 | 主动小范围测试新玩法，产生正向数据后升级为课题 | 随时 |

## 4. 课题筛选标准

每个课题在正式立项前需通过以下筛选：

| 维度 | 评估问题 | 通过标准 |
|------|---------|---------|
| 资源需求 | 需要多少人力、时间、预算？ | ≤ 2周工时 |
| 见效周期 | 多久能看到数据结果？ | ≤ 4周 |
| 可复制性 | 这个玩法是否可以复用到其他场景？ | 是 |
| 影响范围 | 如果成功，对哪个指标有正向影响？ | 明确 |
| 优先级 | 在现有课题中优先级如何？ | 高优先（见下） |

**优先级评估矩阵：**

| | 见效快 | 见效慢 |
|-|--------|--------|
| **影响大** | 🔥🔥🔥 高优先 | 🔥🔥 中等优先 |
| **影响小** | 🔥 中等优先 | ❄️ 低优先/搁置 |

## 5. 课题执行流程

### 阶段一：课题立项（1-2天）
1. 填写《课题立项表》（见附录）
2. 明确假设内容、验证指标、成功标准
3. 提交营销实验室负责人确认

### 阶段二：验证设计（3-5天）
1. 设计验证方案（AB测试 / 小范围试跑 / 数据对比）
2. 确定测试组与对照组（如适用）
3. 制定数据追踪指标（点击率/转化率/ROI等）
4. 预估验证周期（通常2-4周）

### 阶段三：执行验证（2-4周）
1. 按方案执行，期间记录关键数据
2. 每周固定时间（如周五）检视数据进展
3. 如发现异常或方向偏差，及时调整方案

### 阶段四：复盘与结论（3-5天）
1. 汇总验证周期内的所有数据
2. 对比假设与实际结果
3. 判断是否达到成功标准

### 阶段五：固化与培训（1-2周）
1. 如验证成功，编写 SOP 文档
2. 安排团队培训（纳入考核指标）
3. 如验证失败，归档记录，标注原因，作为后续参考

## 7. 成果固化标准

当课题验证成功后，按以下标准进行固化：

| 成果类型 | 要求 |
|---------|------|
| SOP 文档 | 包含背景、适用范围、操作步骤、注意事项、常见问题 |
| 团队培训 | 至少1场，覆盖目标部门，时长≥30分钟 |
| 效果追踪 | 上线后持续追踪2周，确认效果稳定 |

## 8. 考核指标

营销工程师在「假设验证与 SOP 创新」方向的考核：

| 指标 | 说明 |
|------|------|
| 课题数量 | 每季度至少完成 X 个课题（待定） |
| 落地数量 | 每季度至少 X 个课题验证成功并固化 |
| 效果指标 | 落地方案需有明确的数据改善证据 |
| 培训场数 | 每落地1个 SOP，需安排对应团队培训 |
